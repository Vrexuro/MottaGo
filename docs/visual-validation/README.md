# Visual Validation

## Tujuan

Visual Validation merupakan tahap setelah Phase 3 (Design) dan sebelum Phase 4 (Build).

Tujuan tahap ini adalah memvalidasi bahwa seluruh keputusan desain yang telah dibekukan pada:

* Phase 3A — UX Architecture
* Phase 3B — Design System & Frontend Planning
* Phase 3C — Visual Design Specification (Design Freeze)

dapat diterjemahkan menjadi tampilan antarmuka yang sesuai dengan visi produk MottaGo.

Tahap ini tidak mengubah requirement, workflow, maupun scope proyek.

Visual Validation digunakan untuk mengevaluasi kualitas desain sebelum implementasi dimulai.

## Validation Rules

Seluruh desain visual harus:

* Mengikuti PRD yang telah disetujui.
* Mengikuti seluruh Decision Log yang telah ditutup.
* Mengikuti UX Architecture Phase 3A.
* Mengikuti Design System Phase 3B.
* Mengikuti Design Freeze Phase 3C.

Tidak diperbolehkan:

* Menambah fitur baru.
* Mengubah workflow.
* Mengubah navigation structure.
* Mengubah screen inventory.
* Mengubah component inventory.
* Membuat alternatif tema baru.

---

## Review Criteria

Setiap desain yang dihasilkan harus dievaluasi berdasarkan:

1. Konsistensi terhadap Design Freeze.
2. Keterbacaan informasi.
3. Kejelasan KPI dan status operasional.
4. Aksesibilitas.
5. Responsivitas.
6. Kesesuaian dengan karakter Bumi Tenang + Daun Segar.
7. Kesiapan untuk implementasi frontend.
