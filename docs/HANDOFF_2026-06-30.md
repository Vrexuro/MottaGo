## 3. Setup Environment di Device Baru

### Prerequisites

```
Node.js LTS
Git
VS Code (atau editor pilihan)
```

### Clone dan Install

```bash
git clone <repository-url>
cd MottaGo/frontend
npm install
```

### Environment Variables

Salin `.env.example` ke `.env`:

```bash
cp frontend/.env.example frontend/.env
```

Isi dengan credentials Supabase nyata (diperoleh dari Supabase Dashboard):

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

**PENTING:** Credentials nyata belum dikonfigurasi. Dev server akan berjalan tetapi semua panggilan Supabase akan gagal sampai `.env` diisi.

### Menjalankan Dev Server

```bash
cd frontend
npm run dev
```

### Verifikasi Build

```bash
cd frontend
npm run lint    # Harus 0 warning
npm run build   # Harus berhasil tanpa error TypeScript
```

## 5. Database — Status LOCKED

**Database Foundation sudah LOCKED.** Tidak boleh ada perubahan migration tanpa approval eksplisit dari user.

### Migration Files (`supabase/migrations/`)

| File | Konten | Status |
|---|---|---|
| `0001_extensions.sql` | Extension declarations (uuid-ossp, pg_trgm) | LOCKED |
| `0002_tables.sql` | 8 tables + pickup_seq sequence | LOCKED |
| `0003_constraints.sql` | 12 FK + 14 CHECK + 2 UNIQUE | LOCKED |
| `0004_indexes.sql` | 13 performance indexes | LOCKED |
| `0005_triggers.sql` | 4 functions + 6 triggers | LOCKED |
| `0006_seed.sql` | Dev seed data — 26 rows, idempotent | LOCKED |

### Tables

```
vendors            — Vendor pengangkut sampah (SERIAL PK)
stores             — Data toko/restoran (GENERATED IDENTITY PK)
profiles           — User profiles linked ke auth.users (UUID PK)
pickups            — Pickup request (TEXT PK: 'PU-YYMM-NNN')
waste_items        — Item limbah (UUID PK + 2 GENERATED columns)
capacity_snapshots — Snapshot kapasitas (UUID PK + 2 GENERATED columns)
notifications      — Notifikasi in-app (UUID PK)
capacity_alerts    — Alert kapasitas (UUID PK)
```

### Generated Columns (JANGAN masukkan dalam INSERT)

```sql
waste_items.unit         -- GENERATED dari waste_type
waste_items.quantity_kg  -- GENERATED dari quantity + unit
capacity_snapshots.pct_used  -- GENERATED: current_kg/max_kg*100
capacity_snapshots.status    -- GENERATED: 'normal'|'perlu-perhatian'|'kritis'
```

### Trigger Functions

| Function | Trigger | Keterangan |
|---|---|---|
| `fn_set_updated_at` | 3 triggers (profiles, stores, vendors) | Auto-update `updated_at` |
| `fn_generate_pickup_id` | `trg_pickups_generate_id` | Generate `PU-YYMM-NNN` |
| `fn_create_capacity_alert` | `trg_capacity_snapshots_alert` | Auto-create alert + notifikasi |
| `fn_handle_new_user` | `trg_auth_users_new_profile` | Auto-create profile dari auth |

### Business Rules Kritis

- **DL-03**: Hanya 1 pickup aktif per store (enforced via partial UNIQUE index `uix_pickups_one_active_per_store`)
- **DL-02**: `waste_items.unit` di-GENERATE dari `waste_type` — jangan hardcode
- **DL-04**: Notifikasi via polling 30 detik — bukan WebSocket

## 7. Design System

**Aturan kritis:** Selalu gunakan Tailwind aliases dari `tailwind.config.ts`. JANGAN pernah hardcode hex values.

```
Contoh BENAR:  className="bg-brand-primary text-text-secondary"
Contoh SALAH:  className="bg-[#007AFF] text-[#6B7280]"
```

### Threshold Kapasitas (DL-04)

```
Normal        : pct_used < 60%   → --color-capacity-normal   → bg-capacity-normal
Perlu Perhatian: 60% ≤ pct < 90% → --color-capacity-warning  → bg-capacity-warning
Kritis        : pct_used ≥ 90%   → --color-capacity-critical → bg-capacity-critical
```

## 9. Deployment Prerequisites (Sebelum C1)

Sebelum mengintegrasikan authentication nyata:

1. **Buat Supabase Project** via supabase.com
2. **Aktifkan Extensions** di Supabase Dashboard > Database > Extensions:
   - `uuid-ossp`
   - `pg_trgm` ← wajib sebelum migration 0004 dijalankan (WARN-01)
3. **Jalankan Migrations** via Supabase Dashboard > SQL Editor (urutan: 0001 → 0002 → 0003 → 0004 → 0005 → 0006)
4. **Isi `.env`** dengan URL dan anon key dari Supabase project settings
5. **Konfigurasi Supabase Auth** — Email auth enabled, redirect URLs dikonfigurasi
