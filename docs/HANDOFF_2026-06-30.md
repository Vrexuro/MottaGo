## 1. Ringkasan Project

MottaGo adalah sistem manajemen pengurangan limbah makanan untuk restoran. Sistem mencakup pencatatan, klasifikasi, monitoring kapasitas, pelacakan pickup, pelaporan, dan analitik.

**Stakeholder aktif:** Manajer, Pegawai Utility

## 3. Setup Environment di Device Baru

### Prerequisites

```
Node.js LTS
Git
VS Code (atau editor pilihan)
```

### Clone dan Install

```bash
git clone <repository-url>
cd MottaGo/frontend
npm install
```

### Environment Variables

Salin `.env.example` ke `.env`:

```bash
cp frontend/.env.example frontend/.env
```

Isi dengan credentials Supabase nyata (dari Supabase Dashboard):

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

### Menjalankan Dev Server

```bash
cd frontend
npm run dev
```

### Verifikasi Build

```bash
cd frontend
npm run lint    # Harus 0 warning
```

### Tables

```
vendors            — Vendor pengangkut sampah (SERIAL PK) — Master Data only
stores             — Data toko/restoran (GENERATED IDENTITY PK)
store_vendor_assignments — Vendor assignment per store per waste_category (DL-06)
profiles           — User profiles linked ke auth.users (UUID PK)
pickups            — Pickup request (TEXT PK: 'PU-YYMM-NNN')
waste_items        — Item limbah (UUID PK + 2 GENERATED columns)
capacity_snapshots — Snapshot kapasitas (UUID PK + 2 GENERATED columns)
notifications      — Notifikasi in-app (UUID PK)
capacity_alerts    — Alert kapasitas (UUID PK)
```

### Generated Columns (JANGAN masukkan dalam INSERT)

```sql
waste_items.unit         -- GENERATED dari waste_type
waste_items.quantity_kg  -- GENERATED dari quantity + unit
capacity_snapshots.pct_used  -- GENERATED: current_kg/max_kg*100
capacity_snapshots.status    -- GENERATED: 'normal'|'perlu-perhatian'|'kritis'
```

### Business Rules Kritis

- **DL-03** *(revised 2 Juli 2026)*: Hanya 1 pickup aktif per store **per waste_category** — enforced via partial UNIQUE index `(store_id, waste_category) WHERE status IN ('waiting','in-transit')`
- **DL-02**: `waste_items.unit` di-GENERATE dari `waste_type` — jangan hardcode
- **DL-04**: Notifikasi via polling 30 detik — bukan WebSocket

## 7. Design System

**Aturan kritis:** Selalu gunakan Tailwind aliases dari `tailwind.config.ts`. JANGAN pernah hardcode hex values.

```
Contoh BENAR:  className="bg-brand-primary text-text-secondary"
Contoh SALAH:  className="bg-[#007AFF] text-[#6B7280]"
```

### Threshold Kapasitas (DL-04)

```
Normal          : pct_used < 60%   → bg-capacity-normal
Perlu Perhatian : 60% ≤ pct < 90% → bg-capacity-warning
Kritis          : pct_used ≥ 90%   → bg-capacity-critical
```

### Design System Files

```

frontend/src/design-system/globals.css     — Global reset + base styles
```
