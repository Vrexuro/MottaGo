# MottaGo Prototype

## Overview

Prototype ini merupakan representasi visual dari fitur utama sistem **MottaGo – Food Waste Reduction Management System**.

Prototype dikembangkan setelah proses Design Freeze selesai dan digunakan untuk memvalidasi implementasi visual sebelum memasuki tahap pembangunan aplikasi (Phase 4 – Build).

Prototype dibuat berdasarkan dokumen yang telah disetujui pada fase sebelumnya dan tidak menambahkan requirement, workflow, maupun fitur baru di luar ruang lingkup proyek.

## Available Screens

### SCR-M-01 — Manager Dashboard

Tujuan:

Memberikan ringkasan kondisi operasional pengelolaan waste pada restoran.

Fitur utama:

* KPI Overview
* Capacity Monitoring
* Pickup Monitoring
* Waste Trend Analysis
* Quick Actions
* Active Pickup Table

Status:

Approved

---

### SCR-M-02 — Monitoring Kapasitas

Tujuan:

Membantu manajer memantau kapasitas penyimpanan waste dan mendeteksi risiko over-capacity.

Fitur utama:

* Capacity Overview
* Linear Capacity Gauge
* Capacity Trend Analysis
* Waste Category Breakdown
* Threshold Monitoring
* Capacity Alert History

Status:

Approved

---

### SCR-M-04 — Request Pickup

Tujuan:

Memungkinkan manajer mengajukan permintaan pengambilan waste kepada vendor pengelola limbah.

Fitur utama:

* Pickup Request Form
* Capacity Context Information
* Vendor Selection
* Pickup Scheduling
* Request Summary
* Validation & Confirmation States

Status:

Approved
