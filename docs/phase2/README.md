# Phase 2 - Specify

## Tujuan

Fase Specify bertujuan mengubah requirement yang telah disetujui menjadi spesifikasi yang siap digunakan untuk desain dan implementasi.

## Deliverables

* User Stories
* Acceptance Criteria
* Use Cases
* Feature Specifications
* Module Specifications
* Dependency Mapping

## Rules

* Setiap User Story harus memiliki hubungan dengan requirement pada Phase 1.
* Setiap Acceptance Criteria harus dapat diverifikasi.
* Tidak diperbolehkan membuat desain UI atau implementasi teknis pada fase ini.
